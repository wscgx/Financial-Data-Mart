### 2.2 ETL 脚本设计

#### etl_ods_to_delta.py（ODS 层）

```python
"""
MySQL ODS 表 → Delta Lake (MinIO)
全量同步，每日执行
"""
from configs.spark_config import (
    get_spark_session, read_mysql_table, write_delta_table
)

ODS_TABLES = [
    "ods_customer",
    "ods_account",
    "ods_product",
    "ods_transaction",
    "ods_holding",
    "ods_risk_assessment",
]

def run():
    spark = get_spark_session("ETL_ODS_to_Delta")

    for table in ODS_TABLES:
        df = read_mysql_table(spark, table)
        write_delta_table(
            df,
            path=f"s3a://financial-data-lake/ods/{table}",
            mode="overwrite",
            partition_by=None
        )
        print(f"[OK] {table}: {df.count()} rows → Delta Lake")

    spark.stop()
```

#### etl_dwd_to_delta.py（DWD 层）

```python
"""
Delta Lake ODS → Delta Lake DWD
使用 PySpark SQL 实现清洗转换
"""
DWD_ETL_SQL = {
    "dwd_customer_asset_daily": """
        SELECT
            t.transaction_date AS stat_date,
            t.customer_id,
            t.account_id,
            SUM(CASE WHEN t.transaction_type = 'holding' THEN t.amount ELSE 0 END) AS holding_market_value,
            SUM(CASE WHEN t.transaction_type = 'purchase' THEN t.amount ELSE 0 END) AS purchase_amount,
            SUM(CASE WHEN t.transaction_type = 'redemption' THEN t.amount ELSE 0 END) AS redemption_amount,
            'ods_transaction' AS dm_src_info,
            CURRENT_TIMESTAMP() AS etl_load_time
        FROM delta.`s3a://financial-data-lake/ods/ods_transaction` t
        GROUP BY t.transaction_date, t.customer_id, t.account_id
    """,
    # ... 其他 DWD 表的 SQL
}
```

#### etl_ads_sync_mysql.py（ADS → MySQL 回写）

```python
"""
Delta Lake ADS → MySQL
将分析层结果回写 MySQL 供 Streamlit 读取
"""
ADS_TABLES = [
    "ads_executive_dashboard",
    "ads_customer_aum_daily",
    "ads_branch_sales_ranking",
    # ... 全部 ADS 表
]

def run():
    spark = get_spark_session("ETL_ADS_Sync_MySQL")

    for table in ADS_TABLES:
        df = spark.read.format("delta").load(
            f"s3a://financial-data-lake/ads/{table}"
        )
        (
            df.write
            .format("jdbc")
            .option("url", MYSQL_URL)
            .option("dbtable", table)
            .options(**MYSQL_PROPERTIES)
            .mode("overwrite")
            .save()
        )
        print(f"[OK] {table}: synced to MySQL")

    spark.stop()
```

### 2.3 创建 Airflow DAG

`docker/airflow/dags/financial_etl_dag.py`：

```python
from airflow import DAG
from airflow.providers.apache.spark.operators.spark_submit import SparkSubmitOperator
from datetime import datetime

default_args = {
    "owner": "financial_dw",
    "retries": 2,
    "start_date": datetime(2026, 1, 1),
}

with DAG(
    dag_id="financial_etl_pipeline",
    default_args=default_args,
    schedule_interval="0 2 * * *",  # 每日凌晨2点
    catchup=False,
    tags=["financial", "etl"],
) as dag:

    # ODS: MySQL → Delta Lake
    ods_sync = SparkSubmitOperator(
        task_id="ods_mysql_to_delta",
        application="/opt/airflow/dags/scripts/etl_ods_to_delta.py",
        conn_id="spark_default",
        application_args=["--date", "{{ ds }}"],
    )

    # DWD: Delta ODS → Delta DWD
    dwd_build = SparkSubmitOperator(
        task_id="dwd_delta_to_delta",
        application="/opt/airflow/dags/scripts/etl_dwd_to_delta.py",
        conn_id="spark_default",
        application_args=["--date", "{{ ds }}"],
    )

    # DWS: Delta DWD → Delta DWS
    dws_build = SparkSubmitOperator(
        task_id="dws_delta_to_delta",
        application="/opt/airflow/dags/scripts/etl_dws_to_delta.py",
        conn_id="spark_default",
        application_args=["--date", "{{ ds }}"],
    )

    # ADS: Delta DWS → Delta ADS
    ads_build = SparkSubmitOperator(
        task_id="ads_delta_to_delta",
        application="/opt/airflow/dags/scripts/etl_ads_to_delta.py",
        conn_id="spark_default",
        application_args=["--date", "{{ ds }}"],
    )

    # ADS → MySQL 回写
    ads_sync = SparkSubmitOperator(
        task_id="ads_sync_to_mysql",
        application="/opt/airflow/dags/scripts/etl_ads_sync_mysql.py",
        conn_id="spark_default",
    )

    ods_sync >> dwd_build >> dws_build >> ads_build >> ads_sync
```

### 2.4 DAG 依赖链

```
ods_sync → dwd_build → dws_build → ads_build → ads_sync
(MySQL→Δ)  (ΔODS→ΔDWD) (ΔDWD→ΔDWS) (ΔDWS→ΔADS) (ΔADS→MySQL)
```

## 技术选型对比

| 组件 | 当前方案 | 迁移后方案 | 理由 |
|------|---------|-----------|------|
| 存储 | MySQL (InnoDB) | Delta Lake (MinIO) | 支持 ACID、时间旅行、Schema Evolution |
| 批处理 | MySQL SQL | PySpark + Delta Lake | 分布式计算，支持 TB 级数据 |
| 流处理 | 无 | Flink / Spark Streaming | 实时聚合，毫秒级延迟 |
| 调度 | 手动执行 | Airflow | DAG 编排，依赖管理，重试机制 |
| 消息队列 | 无 | Kafka | 解耦数据源，支持背压 |
| 监控 | 无 | Prometheus + Grafana | 全链路可观测 |
| 数据质量 | SQL 手动校验 | Great Expectations | 自动化、可配置、可追溯 |

## 风险与注意事项

1. **内存要求**：Spark + Flink + Kafka 至少需要 8GB 可用内存
2. **磁盘空间**：Delta Lake 数据存储在 MinIO，预留 50GB+
3. **兼容性**：保持 MySQL 作为查询层，确保 Streamlit 无需大改
4. **渐进迁移**：Phase 2 完成后，旧 MySQL ETL 脚本保留，可随时回退
5. **数据一致性**：ADS 回写 MySQL 时使用 overwrite 模式，避免数据重复
