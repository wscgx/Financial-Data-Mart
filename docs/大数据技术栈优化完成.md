## Phase 7: 数据质量框架 ✅

### 7.1 校验规则引擎
- `data_quality/run_validation.py`: 轻量级数据质量校验脚本
- 支持完整性、唯一性、一致性、时效性四类校验
- 输出 CSV 校验报告到 `data/quality_reports/`

### 7.2 校验规则清单
| 规则名称 | 校验类型 | 说明 |
|---------|---------|------|
| ods_customer_completeness | 完整性 | ODS客户表主键非空 |
| ods_customer_uniqueness | 唯一性 | ODS客户表主键唯一 |
| ods_transaction_completeness | 完整性 | ODS交易表主键非空 |
| dwd_asset_daily_completeness | 完整性 | DWD资产日快照主键非空 |
| dwd_asset_daily_consistency | 一致性 | DWD资产日快照无负值 |
| dws_value_profile_completeness | 完整性 | DWS客户价值画像主键非空 |
| ads_dashboard_completeness | 完整性 | ADS高管驾驶舱主键非空 |
| ads_aum_daily_timeliness | 时效性 | ADS AUM数据7天内有效 |

## 注意事项

1. **内存要求**: Spark + Flink + Kafka 至少需要 8GB 可用内存
2. **磁盘空间**: Delta Lake 数据存储在 MinIO，预留 50GB+
3. **首次启动**: 需要等待所有服务初始化完成 (约 2-3 分钟)
4. **数据初始化**: 首次使用需运行 ETL 脚本初始化数据
