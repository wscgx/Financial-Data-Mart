# AI智能财富分析平台 - 标准作业流程 (SOP)

## 触发条件
用户提供业务需求输入。

## 执行流程

### Phase 1: 需求分析
- **任务**：深入理解业务背景、分析目标、数据来源、分析维度与粒度
- **输出**：需求分析摘要（业务目标、核心指标、数据范围、分析维度）
- **检查点**：与用户确认需求理解无误，不急于落地代码

### Phase 2: 数仓模型设计
- **任务**：基于 Kimball 维度建模思想，设计 ODS → DWD → DWS → ADS 分层模型
- **输出**：模型设计文档（主题域划分、事实表/维度表设计、粒度、主键、关联关系、数据流）
- **检查点**：与用户确认模型设计合理，遵循项目命名规范

### Phase 3: DDL生成与建表
- **任务**：自动生成各层 DDL 语句，包含完整中文注释
- **输出**：`sql/ddl/` 目录下的建表脚本，并自动执行到 MySQL 数据库
- **规范**：
  - 表名：`{层级前缀}_{主题域}_{业务含义}_{粒度/频率}`
  - 必须包含中文表注释和字段注释
  - 主键/分区键明确

### Phase 4: ETL脚本生成与数据加载
- **任务**：生成 ODS → DWD → DWS → ADS 的 ETL SQL
- **输出**：`etl/` 目录下的 ETL 脚本，并自动执行加载到 MySQL
- **规范**：
  - 支持增量/全量加载策略
  - 避免数据倾斜，合理使用窗口函数
  - 脚本顶部注明数据来源与目标表

### Phase 5: 数据质量校验（ADS层）
- **任务**：对 ADS 层数据进行完整性、唯一性、准确性、一致性、勾稽关系校验
- **输出**：`sql/quality_check/` 下的校验脚本及校验报告
- **检查点**：**校验必须全部通过**。若未通过，自动返回 Phase 4 修复，严禁跳过

### Phase 6: 指标体系生成
- **任务**：基于 ADS 层数据与模型，自动生成指标体系文档
- **输出**：指标字典（指标名称、业务口径、计算逻辑、业务意义、更新频率、数据源表）

### Phase 7: BI方案设计（本地文件，待确认）
- **任务**：设计可视化方案、图表选型、看板布局、交互逻辑
- **输出**：`docs/bi_proposal.md` 文件，包含完整 BI 方案说明
- **检查点**：**必须等待用户明确回复"确认"或"通过"后，才可进入下一阶段**

### Phase 8: 前端分析报告输出
- **任务**：根据确认的 BI 方案，更新 Streamlit 前端代码或生成分析报告
- **输出**：更新 `app.py` 或生成静态报告，前端服务自动刷新展示
- **检查点**：与用户确认展示效果，支持迭代调整

## 强制规则（Agent 必须遵守）
1. **分步执行**：严禁跨阶段或一次性输出所有结果，必须按 Phase 1 → 8 顺序推进
2. **质量门禁**：Phase 5 数据质量校验不通过，绝对禁止进入 Phase 6
3. **人工确认**：Phase 7 输出 BI 方案后，**必须暂停等待用户确认**，未经确认不得修改前端代码
4. **注释规范**：所有 SQL/DDL/ETL 必须包含完整中文注释
5. **规范遵循**：严格遵循项目 `AGENTS.md` 中的命名规范、分层规范与编码规范
6. **文件落地**：所有中间产物（DDL、ETL、校验脚本、BI方案）必须落地到对应目录，供用户查阅
