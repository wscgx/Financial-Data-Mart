# 数据架构师进阶学习指南

> **目标定位**：从高级数据工程师向数据架构师转型
> **适用背景**：7 年开发经验（含 Java 基础 + 4 年数据仓库/大数据开发）

---

## 🏛️ 一、 核心能力图谱（架构师视角）

| 能力维度 | 开发关注点 | **架构师关注点** |
| :--- | :--- | :--- |
| **技术选型** | 哪个工具好用？怎么写代码？ | **ROI 分析**：业务场景匹配度、社区活跃度、运维成本、团队学习曲线、未来 3-5 年技术演进趋势。 |
| **数据建模** | 表怎么建？SQL 怎么写？ | **企业级模型**：主题域划分、总线矩阵、一致性维度、指标体系标准化（OneData）、模型复用率。 |
| **数据治理** | 数据错了怎么修？ | **治理体系**：数据标准、质量 SLA、元数据管理、血缘追踪、安全合规、生命周期管理、**数据资产目录**。 |
| **性能与成本** | SQL 怎么优化跑得快？ | **架构调优 & FinOps**：资源隔离、弹性伸缩、存储计算分离、冷热数据分层、集群成本优化（降本增效）。 |
| **架构模式** | Lambda/Kappa 架构怎么用？ | **湖仓一体 (Lakehouse)**、**Data Mesh (数据网格)**、**Data Fabric (数据编织)**、云原生数据架构。 |

## 🚀 二、 技术栈升级路径

### 1. 实时湖仓架构 (Real-time Lakehouse)
- **核心技术**：Apache Flink, Apache Iceberg / Hudi / Paimon, StarRocks / Doris / ClickHouse
- **学习目标**：
  - 掌握 **流批一体架构**：Flink 批流一体原理与实践。
  - 深入 **数据湖技术**：理解 Upsert、Time Travel、Schema 演进、小文件合并机制。
  - 掌握 **OLAP 引擎原理**：向量化执行、列式存储、索引机制（Bitmap/倒排）、MPP 架构。
  - **架构设计实践**：设计基于 `Iceberg + Flink + StarRocks/Doris` 的实时湖仓架构，解决数据延迟、一致性或存储成本问题。

### 2. 云原生与 DataOps
- **核心技术**：Kubernetes (K8s), Docker, Serverless, CI/CD (GitLab CI, Jenkins)
- **学习目标**：
  - **云原生大数据**：K8s 上运行 Spark/Flink (K8s on Hadoop)、存算分离架构（对象存储 S3/OSS + 计算引擎）。
  - **DataOps 实践**：数据开发的 CI/CD、自动化测试、版本控制、环境管理、数据 Pipeline 编排。

### 3. 底层原理与源码级掌控
- **核心技术**：Spark/Flink 源码, JVM 调优, 存储格式 (Parquet/ORC)
- **学习目标**：
  - **计算引擎原理**：内存管理、Shuffle 机制、Checkpoint 原理、反压机制。能回答"为什么 OOM？""为什么数据倾斜？""如何自定义 Source/Sink？"。
  - **存储格式**：Parquet/ORC 的文件结构、编码压缩算法。
  - **JVM 调优**：GC 策略、堆内存分配、Native Memory Tracking。

## 🛡️ 三、 数据治理与体系化建设

### 1. 元数据与血缘管理
- **工具**：Apache Atlas, DataHub, OpenMetadata
- **目标**：
  - 掌握业务元数据、技术元数据、操作元数据的采集与管理。
  - 实现 SQL 血缘自动解析（AST 解析），支持影响分析与根因追溯。

### 2. 数据质量与 SLA
- **目标**：
  - 建立数据质量闭环：规则定义 -> 监控告警 -> 工单流转 -> 修复验证 -> 考核指标。
  - 定义数据 SLA（时效性、准确性、完整性），并与业务考核挂钩。

### 3. 数据安全与合规
- **工具**：Apache Ranger, 动态脱敏插件
- **目标**：
  - 数据分级分类标准制定。
  - 行列级权限管控、动态脱敏、审计日志。
  - 熟悉 GDPR、数据安全法、个人信息保护法相关要求。

### 4. 指标体系与 OneData
- **目标**：
  - 主导从业务需求到原子指标、派生指标、修饰词的完整设计。
  - 保证指标口径一致性，建立指标字典。
  - 熟悉阿里 OneData 体系方法论。

## 📐 四、 架构设计与业务思维

### 1. 技术选型与权衡 (Trade-off)
- **能力**：
  - 不再只追求“技术新”，而是追求“最合适”。
  - 平衡性能、成本、稳定性、可维护性、团队能力。
  - 编写架构设计文档 (ADD)，包含背景、目标、架构图、选型对比、风险评估、演进规划。

### 2. 降本增效 (FinOps)
- **能力**：
  - 资源利用率监控与优化。
  - 冷热数据分层存储策略。
  - 计算资源弹性伸缩与预留实例管理。
  - 数据生命周期管理（TTL、归档、删除）。

### 3. 数据服务化 (Data API)
- **能力**：
  - 设计统一的数据服务层，将数据以 API 形式提供给业务系统。
  - 解耦底层存储与上层应用，提供查询加速、缓存、限流熔断机制。

### 4. 数据价值评估
- **能力**：
  - 如何衡量数据项目的 ROI？
  - 如何推动数据资产化？
  - 数据产品思维：从数据服务角度思考，而非纯技术开发。

## 📚 五、 学习资源与行动建议

### 📖 必读经典书籍
1. **《数据密集型应用系统设计》(DDIA)** —— 架构师圣经，必读。
2. **《大数据之路：阿里巴巴大数据实践》** —— 了解 OneData 体系与企业级数仓建设。
3. **《数据架构师之路》/《企业数据架构》** —— 架构方法论。
4. **《Flink 内核原理与实现》/《Spark 内核设计的艺术》** —— 源码级深入。

### 🎓 权威认证（加分项）
- **CDMP** (Certified Data Management Professional) —— 数据管理国际认证，涵盖治理、建模、质量等。
- **TOGAF** —— 企业架构框架，包含数据架构部分。
- **云厂商认证**：AWS/Aliyun 大数据架构师认证。

### 🛠 工作实践建议
1. **主导技术重构**：例如将老旧的 Hive 数仓迁移到 Iceberg 湖仓，或引入新的 OLAP 引擎替换慢查询。
2. **推动治理落地**：主动发起数据质量提升专项，建立数据标准规范。
3. **编写架构文档**：练习写架构设计文档，提升全局思考与表达能力。
4. **技术社区与影响力**：参与开源项目（提交 PR、修复 Bug），在技术博客/公众号分享架构实践心得。

## 📝 六、 简历优化建议

1. **量化成果**：如“数据质量提升至 99.9%"可补充具体指标定义和衡量方式。
2. **技术分层**：专业技能区分“精通”、“熟练”、“了解”层级。
3. **突出亮点**：补充技术难点攻克的具体案例（如解决过什么复杂的数据倾斜问题、架构重构带来的成本降低比例）。
4. **持续学习**：补充近期学习的技术或参与的开源项目/技术社区。

---

> **总结**：从 **数据开发** 到 **数据架构师**，最大的跨越是 **思维模式** 的转变。
> - 不再只关注“点”（某个任务、某张表），而是关注 **“面”**（整个数据平台、数据流、治理体系）。
> - 不再只追求“技术新”，而是追求 **“最合适”**（平衡性能、成本、稳定性、可维护性）。
> - 从“被动接需求” 变为 **“主动规划数据战略”**，用数据驱动业务。
